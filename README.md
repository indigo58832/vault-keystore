# Vault — менеджер ключей Windows/Office

Программа для управления складом активационных ключей Microsoft. Хранит ключи по категориям, умеет проверять их через локальный сервер `winkeycheck`, помечать как проданные, переносить в архив, искать замену мёртвому ключу из своего пула.

Состоит из двух частей:

| Компонент | Что делает |
|---|---|
| **winkeycheck** (`localhost:17777`) | Локальный HTTP-сервер. Принимает ключ, расшифровывает его, дёргает Microsoft, возвращает тип/эдишн/статус (online OK или код ошибки). |
| **Vault** (Qt-приложение) | GUI поверх SQLite БД. Категории слева, ключи справа, кнопки проверки. Под капотом ходит к `winkeycheck`. |

## Установка

### Linux

```bash
# 1. Зависимости системы
sudo apt install python3 python3-pip python3-pyqt6 xclip wine32

# 2. Клонируй наш репо (тут Vault)
git clone <url> ~/Applications/work_tool
cd ~/Applications/work_tool

# 3. Pip-зависимости
pip3 install requests pynput urllib3

# 4. Внутри work_tool клонируй сервер winkeycheck (публичный форк)
git clone https://github.com/WitherOrNot/winkeycheck

# 5. Wine prefix для winkeycheck (нужен 32-bit)
WINEPREFIX=~/.wine32 WINEARCH=win32 winecfg

# 6. Стартуй сервер
cd winkeycheck && python3 server.py &

# 7. Стартуй Vault
cd ../keystore && ./start.sh
```

### Windows

1. **Поставь Python 3.11+**: https://www.python.org/downloads/  
   ВАЖНО: при установке поставь галку «Add Python to PATH».
2. **Скачай наш репо**: «Code → Download ZIP» на GitHub, распакуй в `C:\work_tool`.
3. **Также скачай winkeycheck**: https://github.com/WitherOrNot/winkeycheck — нужны файлы из этого репо в подпапке `winkeycheck\` рядом с `keystore\`.
4. **Запусти `keystore\install.bat`** — поставит PyQt6, requests, pynput.
5. **Запусти сервер**: двойной клик `winkeycheck\start.bat` — откроется окно с логами, **не закрывай**.
6. **Запусти Vault**: двойной клик `keystore\start.bat` — появится иконка в трее.

> На Windows Wine не нужен — `pidgenx_caller.exe` запускается напрямую.

## Как пользоваться

### Категории и разделы

Слева — дерево категорий, разбитое на **разделы** (Windows, Office, PSN или какие сам создашь).

- **«+ Новая категория»** (внизу) — создаёт категорию. В диалоге укажи имя и раздел (можно вписать новый).
- **Двойной клик** на категории — переименовать.
- **Двойной клик** на разделе — переименовать раздел целиком.
- **Правый клик** на категории — переименовать, удалить, переместить в другой раздел.
- **Правый клик** на разделе — добавить категорию в этот раздел, переименовать, удалить.
- Архив (`Склад старых ключей`) — спец-раздел для мёртвых MAK-ключей. Всегда виден.

### Импорт ключей

1. Выбери категорию слева (или создай).
2. Жми **«+ Добавить ключи»**, вставь список (по одному ключу в строке, регистр любой).
3. Жми OK — ключи добавятся И сразу проверятся через `winkeycheck` (если включена галочка **Consume**).

Галочка **«Consume»** внизу:
- ВКЛ — точный код ошибки от Microsoft, **может потратить активацию** Retail-ключа.
- ВЫКЛ — только локальная расшифровка + MAK count (быстро, без расхода).

### Проверка

- **«Проверить выделенные»** — проверяет ключи, что ты выделил в таблице.
- **«Проверить непроверенные»** — проверяет все ключи в текущей категории со статусом «не проверено».
- **Автопроверка раз в сутки** — фоновая. При старте Vault через минуту запустит первый прогон если давно не было. Проходит **все** категории (даже архив). Мёртвые Retail/OEM удаляются, мёртвые MAK переезжают в архив.

### Действия с ключами (правый клик на ключ или кнопки)

- **«Скопировать ключ(и)»** — в буфер обмена.
- **«Пометить продано»** — статус «→ продан», ключ становится оранжевым.
- **«Вернуть в свободные»** — обратно.
- **«Проверить»** — проверка через `winkeycheck`.
- **«Дублировать (recode)…»** — для MAK: создаёт несколько вариантов исходного ключа (recode-history).
- **«📦 В архив»** — переносит выделенные ключи в архив. Также есть **кнопка «В архив»** под таблицей.
- **«Удалить»** — удаляет навсегда.

### Quick Check (Ctrl+Shift+S)

Глобальный хоткей (если включён в `setup_global_hotkey` или в системных шорткатах GNOME/Windows). Открывает мини-окно поверх всех окон:

- Подхватывает выделенный текст (на Linux — X11 PRIMARY, на Windows — обычный буфер).
- Извлекает ключ(и) regex'ом, сразу проверяет.
- Если ключ из твоего пула оказался мёртвым (003/004/006/060/020) — ищет в БД живой ключ того же продукта и предлагает замену.
- Клик по ключу в результатах — копирует в буфер.

Esc — закрыть окно.

## Структура БД

- **Sections** — разделы (Windows, Office, PSN…).
- **Categories** — категории внутри разделов. У каждой может быть `section_id` или NULL («Без раздела»).
- **Keys** — сами ключи. Поля: `key`, `category_id`, `status` (`available`/`phone_only`/`sold`/`blocked`/`unknown`), `last_checked_at`, `is_mak`, `mak_count`, `online_code`, `description`, `edition`, и т.д.

БД лежит в `~/.local/share/keystore/keys.db` (Linux) или `C:\Users\<имя>\.local\share\keystore\keys.db` (Windows).

Бэкапы делай руками — просто скопируй `.db` файл.

## Известные нюансы

- **Windows**: глобальный хоткей через `pynput` иногда не цепляется без прав администратора. Если Ctrl+Alt+K не работает — назначь шорткат в системных настройках на запуск `start.bat`.
- **Linux**: для PRIMARY selection нужен `xclip` или `xsel`. Если нет — будет браться обычный буфер (как на Windows).
- **Сервер winkeycheck должен быть запущен** перед Vault — иначе проверка не работает (внизу окна светофор «сервер» горит красным).
- **MAK consume жжёт 1 активацию из тысячи** (~копейки). **Retail consume жжёт ВЕСЬ ключ**. Включай Consume осознанно.
